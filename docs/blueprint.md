# **App Name**: SafeCampus AI

## Core Features:

- Anonymous Incident Reporting: Allows students to report safety incidents anonymously without requiring login.
- Authenticated Incident Reporting: Allows students to report incidents and track their status. Requires user authentication.
- AI-Powered Incident Classification: Uses the Gemini API to automatically classify incident types (e.g., harassment, theft).
- AI Risk Level Estimation: Uses the Gemini API to estimate the risk level of each incident (Low, Medium, High, Critical).
- AI Incident Summarization: Uses the Gemini API to generate a concise summary of long incident descriptions.
- Admin Incident Management Dashboard: Provides admins with a dashboard to view, filter, search, and update the status of incident reports.
- User Authentication and Authorization: Implements Firebase Authentication to manage user accounts and roles (student/admin).

## Style Guidelines:

- Primary color: Light blue (#A0D2EB) to convey trust and safety.
- Background color: Off-white (#F5F5F5) for a clean, minimalist feel.
- Accent color: Soft orange (#FFB347) for call-to-action buttons and important alerts.
- Headline font: 'Space Grotesk' sans-serif font for headers to give a modern, slightly techy look. Body font: 'Inter' for the main text.
- Use simple, clear icons from a library like Material Icons to represent incident types and statuses.
- Maintain a clean and organized layout with clear visual hierarchy. Prioritize key information and actions.
- Use subtle animations and transitions to provide feedback and enhance user experience.
